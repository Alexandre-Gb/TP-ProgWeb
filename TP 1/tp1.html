Exercice 3 - Service HTTP

  2. Utilisez la commande grep sur le fichier /etc/services pour déterminer le numéro du port TCP correspondant au service http.

    man /etc/services | grep -i http
    Le numéro du port pour HTTP est le port 80.

  4. A quoi correspondent les 3 valeurs sur la première ligne de l'entête ?

    La première valeur est la version du protocole HTTP utilisé en réponse par le serveur distant.
    La deuxième valeur est le code de retour HTTP (ici, un code 4XX, ce qui indique une erreur due au client).
    La troisième valeur est le message associé au code de retour HTTP.

  5. A quoi correspond la valeur de Content-Type ?

    La valeur de Content-Type correspond au type de contenu du corp (body) de la réponse HTTP.
    Ici, le type de contenu est "text/html", ce qui permet à un navigateur web d'afficher la réponse sous le bon format, et non d'afficher le texte brut à l'écran.

  6. A quoi correspond la valeur de Content-Length ?

    La valeur de Content-Length correspond à la taille du contenu de la réponse HTTP envoyée au client (en octets).

  8. Pourquoi dans la version 1.1 du protocole HTTP, on doit fournir le Host ?

    Dans la version 1.1 du protocole HTTP, on doit fournir le Host pour permettre au serveur de savoir quelle ressource demander.
    En effet, un même serveur peut, depuis cette version, héberger plusieurs sites web. Il doit donc savoir quel site web est demandé par le client.

Conclusion personnelle

Ce premier TP permet de nous familiariser aux bases de l'HTML, du protocole HTTP et de la récolte basique d'informations à partir d'une URL.

Certaines questions, notamment pour l'exercice 4, demandent une mise en page (au travers de CSS) et une structure de page plus soignée.
L'objectif final est d'inciter les étudiants à rendre un code propre, structuré, et suivant les bonnes normes et pratiques de chaque langage utilisé.
